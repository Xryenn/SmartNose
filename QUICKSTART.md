Step 1: Setup Supabase Database
────────────────────────────────────────────────────────────────────────────────

1. Go to: https://app.supabase.com
2. Login/Signup with GitHub
3. Create a new project
   • Project name: smartnose-tpa
   • Region: Singapore (or closest to you)
   • DB password: Generate strong one
4. Wait for project to initialize (5-10 minutes)

5. Go to: Settings → API
6. Copy:
   • Project URL (https://xxxxx.supabase.co)
   • Anon Public Key

7. Create Tables - Go to SQL Editor and run:

   CREATE TABLE sensor_readings (
     id bigint PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
     device_id TEXT NOT NULL,
     ch4_ppm DECIMAL NOT NULL,
     h2s_ppm DECIMAL NOT NULL,
     temperature DECIMAL NOT NULL,
     humidity DECIMAL NOT NULL,
     latitude DECIMAL,
     longitude DECIMAL,
     gps_valid BOOLEAN DEFAULT FALSE,
     alert_ch4 BOOLEAN DEFAULT FALSE,
     alert_h2s BOOLEAN DEFAULT FALSE,
     created_at TIMESTAMP DEFAULT NOW()
   );

   CREATE TABLE alerts_log (
     id bigint PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
     device_id TEXT NOT NULL,
     alert_type TEXT NOT NULL,
     severity TEXT NOT NULL,
     ppm_value DECIMAL NOT NULL,
     created_at TIMESTAMP DEFAULT NOW()
   );

   ALTER TABLE sensor_readings ENABLE ROW LEVEL SECURITY;
   ALTER TABLE alerts_log ENABLE ROW LEVEL SECURITY;

   CREATE POLICY "Allow public read" ON sensor_readings FOR SELECT USING (true);
   CREATE POLICY "Allow public read" ON alerts_log FOR SELECT USING (true);

8. Optional: Enable Realtime
   • Go to: Realtime
   • Enable for sensor_readings & alerts_log tables

Step 6: Connect ESP32 Sensor (Optional but Recommended)

1. Open file: ESP32_Sketch.ino

2. Update WiFi credentials:
   const char* ssid = "YOUR_WIFI_SSID";
   const char* password = "YOUR_WIFI_PASSWORD";

3. Update Vercel URL:
   const char* vercelUrl = "https://YOUR-VERCEL-URL/api/readings";

4. Update sensor pins (if different from default):
   #define CH4_SENSOR_PIN 32
   #define H2S_SENSOR_PIN 33
   etc.

5. Install required libraries:
   • Arduino IDE → Sketch → Include Library → Manage Libraries
   • Search and install: "ArduinoJson" by Benoit Blanchon

6. Upload sketch to ESP32:
   • Connect ESP32 to USB
   • Select Board: ESP32
   • Select Port: COM3 (or your port)
   • Upload

7. Monitor serial output:
   • Open Serial Monitor (Ctrl+Shift+M)
   • Baud rate: 115200
   • Verify ESP32 connects to WiFi and sends data

🆘 COMMON ISSUES & SOLUTIONS:
═════════════════════════════════════════════════════════════════════════════════

❌ "Supabase connection failed"
→ Check .env.local has correct URL and Key
→ Verify Supabase project is running
→ Check browser console for CORS errors

❌ "API endpoint not found"
→ Verify Vercel URL is correct
→ Check api/readings.js was deployed
→ Check Vercel function logs

❌ "Data not showing in dashboard"
→ Verify table sensor_readings exists
→ Manually insert test data
→ Check RLS policies allow read access

❌ "ESP32 won't connect"
→ Check WiFi SSID/password
→ Verify Vercel URL is correct
→ Monitor serial output for errors
→ Install ArduinoJson library
