# Smart Nose TPA v2.0

Real-time Gas Monitoring untuk TPA (Tempat Pembuangan Akhir) dengan sensor CH4, H2S, suhu, kelembapan, dan GPS.

## 🚀 Hosting ke Vercel

### Prasyarat
1. Akun GitHub dengan repository project ini
2. Akun Vercel (https://vercel.com)
3. Akun Supabase (https://supabase.com)

### Langkah 1: Setup Supabase

1. **Buat Project Supabase**
   - Buka https://app.supabase.com
   - Klik "New Project"
   - Pilih region (pilih yang terdekat dengan Indonesia, misal: Singapore)
   - Tunggu project selesai di-setup

2. **Dapatkan Credentials**
   - Di dashboard Supabase, pergi ke: **Settings > API**
   - Copy:
     - **Project URL** (format: `https://xxxxxx.supabase.co`)
     - **anon public** key

3. **Siapkan Database Tables**
   - Di Supabase, buka **SQL Editor**
   - Run query berikut untuk membuat tabel sensor_readings:

```sql
CREATE TABLE sensor_readings (
  id bigint PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
  device_id TEXT NOT NULL,
  ch4_ppm DECIMAL NOT NULL,
  h2s_ppm DECIMAL NOT NULL,
  temperature DECIMAL NOT NULL,
  humidity DECIMAL NOT NULL,
  latitude DECIMAL,
  longitude DECIMAL,
  gps_valid BOOLEAN DEFAULT FALSE,
  alert_ch4 BOOLEAN DEFAULT FALSE,
  alert_h2s BOOLEAN DEFAULT FALSE,
  created_at TIMESTAMP DEFAULT NOW()
);

CREATE TABLE alerts_log (
  id bigint PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
  device_id TEXT NOT NULL,
  alert_type TEXT NOT NULL,
  severity TEXT NOT NULL,
  ppm_value DECIMAL NOT NULL,
  created_at TIMESTAMP DEFAULT NOW()
);

-- Enable Row Level Security (optional tapi disarankan)
ALTER TABLE sensor_readings ENABLE ROW LEVEL SECURITY;
ALTER TABLE alerts_log ENABLE ROW LEVEL SECURITY;

-- Create policy untuk read access
CREATE POLICY "Allow public read access" ON sensor_readings
  FOR SELECT USING (true);

CREATE POLICY "Allow public read access" ON alerts_log
  FOR SELECT USING (true);
```

### Langkah 2: Deploy ke Vercel

#### Option A: Via GitHub (Recommended)

1. **Push ke GitHub**
   ```bash
   git init
   git add .
   git commit -m "Initial commit: Smart Nose TPA"
   git branch -M main
   git remote add origin https://github.com/USERNAME/smartnose2.git
   git push -u origin main
   ```

2. **Connect ke Vercel**
   - Buka https://vercel.com/import
   - Login dengan GitHub
   - Select repository `smartnose2`
   - Klik "Import"

3. **Set Environment Variables**
   - Di halaman project settings, pergi ke: **Settings > Environment Variables**
   - Tambahkan:
     - `SUPABASE_URL` = URL Supabase Anda
     - `SUPABASE_ANON_KEY` = Anon Key Supabase Anda
   - Klik "Save"

4. **Deploy**
   - Klik tombol "Deploy"
   - Tunggu hingga selesai (~2-3 menit)
   - Setelah selesai, Anda akan mendapat URL: `https://your-project.vercel.app`

#### Option B: Via Vercel CLI

1. **Install Vercel CLI**
   ```bash
   npm i -g vercel
   ```

2. **Deploy**
   ```bash
   vercel
   ```
   - Follow prompts
   - Konfigurasi environment variables saat diminta

### Langkah 3: Update Konfigurasi di Aplikasi

Setelah di-deploy ke Vercel:

1. **Buka aplikasi di browser**
   - URL: `https://your-project.vercel.app`

2. **Pergi ke Settings (⚙️)**
   - Klik tab Settings
   - Di bagian "Konfigurasi Supabase", pastikan URL dan Key sudah benar
   - Atau update file `.env.local` jika development

### Langkah 4: Connect Sensor ESP32

Untuk ESP32 sensor, update konfigurasinya agar mengirim data ke:
```
POST https://your-project.vercel.app/api/readings
```

Atau langsung ke Supabase dengan credentials yang sama.

## 🔒 Security Best Practices

1. **Jangan commit `.env.local`** - Sudah di `.gitignore`
2. **Gunakan RLS (Row Level Security)** di Supabase untuk proteksi data
3. **Rotate API keys** secara berkala di Supabase
4. **Gunakan service role key hanya di backend** (jangan di frontend)
