# 📋 Smart Nose TPA - Deployment Checklist

Checklist lengkap untuk deployment ke Vercel dan connect ke Supabase.

## 🔧 Persiapan Awal

- [ ] **Daftar Vercel** → https://vercel.com/signup
- [ ] **Daftar Supabase** → https://app.supabase.com
- [ ] **Git setup** (kalau belum):
  ```bash
  git config --global user.name "Your Name"
  git config --global user.email "your@email.com"
  ```

---

## 📚 Supabase Setup

### Database Creation

- [ ] Login ke Supabase dashboard
- [ ] Create new project dengan region Singapore (paling dekat dengan Indonesia)
- [ ] Tunggu project selesai di-setup (±5 menit)

### Create Tables

- [ ] Buka **SQL Editor**
- [ ] Run SQL script untuk membuat tabel:
  ```sql
  CREATE TABLE sensor_readings (
    id bigint PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
    device_id TEXT NOT NULL,
    ch4_ppm DECIMAL NOT NULL,
    h2s_ppm DECIMAL NOT NULL,
    temperature DECIMAL NOT NULL,
    humidity DECIMAL NOT NULL,
    latitude DECIMAL,
    longitude DECIMAL,
    gps_valid BOOLEAN DEFAULT FALSE,
    alert_ch4 BOOLEAN DEFAULT FALSE,
    alert_h2s BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMP DEFAULT NOW()
  );

  CREATE TABLE alerts_log (
    id bigint PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
    device_id TEXT NOT NULL,
    alert_type TEXT NOT NULL,
    severity TEXT NOT NULL,
    ppm_value DECIMAL NOT NULL,
    created_at TIMESTAMP DEFAULT NOW()
  );

  ALTER TABLE sensor_readings ENABLE ROW LEVEL SECURITY;
  ALTER TABLE alerts_log ENABLE ROW LEVEL SECURITY;

  CREATE POLICY "Allow public read access" ON sensor_readings FOR SELECT USING (true);
  CREATE POLICY "Allow public read access" ON alerts_log FOR SELECT USING (true);
  ```

### Get API Credentials

- [ ] Pergi ke **Settings > API**
- [ ] Copy **Project URL** (format: `https://xxxxx.supabase.co`)
- [ ] Copy **anon public key** (panjang string dari field "anon public")
- [ ] Simpan di tempat aman (notepad/password manager)

### Enable Realtime (Optional tapi Recommended)

- [ ] Di Supabase, pergi ke **Realtime**
- [ ] Klik **Enable realtime** untuk tabel `sensor_readings`
- [ ] Klik **Enable realtime** untuk tabel `alerts_log`

## 🔑 Environment Variables

### Di Local Machine

- [ ] Buka file `.env.local` di project folder
- [ ] Isi dengan Supabase credentials:
  ```
  SUPABASE_URL=https://xxxxx.supabase.co
  SUPABASE_ANON_KEY=your_anon_key_here
  ```
- [ ] **JANGAN commit file ini** (sudah di .gitignore)

## 🚀 GitHub & Deployment

### Push ke GitHub

- [ ] **Create repository** di GitHub:
  - Repository name: `smartnose2`
  - Description: "Real-time Gas Monitoring untuk TPA"
  - Visibility: Public (atau Private)
  - Click "Create repository"

- [ ] **Di local folder smartnose2**, jalankan:
  ```bash
  git init
  git add .
  git commit -m "Initial commit: Smart Nose TPA"
  git branch -M main
  git remote add origin https://github.com/YOUR_USERNAME/smartnose2.git
  git push -u origin main
  ```

- [ ] **Verify** di GitHub bahwa semua files sudah ter-push:
  - Visit https://github.com/YOUR_USERNAME/smartnose2
  - Pastikan ada file: index.html, manifest.json, package.json, vercel.json

### Deploy ke Vercel

- [ ] Login ke Vercel
- [ ] Dashboard → **Add New** → **Project**
- [ ] Import GitHub repository `smartnose2`
- [ ] **Environment Variables**:
  - Click "Add Environment Variables"
  - Add `SUPABASE_URL` = (Supabase URL Anda)
  - Add `SUPABASE_ANON_KEY` = (Anon key Supabase Anda)
- [ ] Click **Deploy**
- [ ] Tunggu hingga status berubah menjadi "Ready" (±2-3 menit)

### Dapatkan Vercel URL

- [ ] Setelah deployment selesai, copy URL yang diberikan
- [ ] Format: `https://smartnose2-xxxxxxx.vercel.app`
- [ ] Test di browser → pastikan aplikasi loading

## 🔗 Connect ESP32

### Update Sensor Firmware

- [ ] Open `ESP32_Sketch.ino` di Arduino IDE
- [ ] Update variables:
  ```cpp
  const char* ssid = "YOUR_WIFI_SSID";
  const char* password = "YOUR_WIFI_PASSWORD";
  const char* vercelUrl = "https://your-vercel-domain.vercel.app/api/readings";
  const char* device_id = "TPA_01";
  ```
- [ ] Update sensor pin configuration sesuai wiring Anda
- [ ] Upload ke ESP32

### Test Sensor Data Flow

- [ ] Power on ESP32
- [ ] Check serial monitor (baud rate 115200)
- [ ] Pastikan:
  - ✓ WiFi connect
  - ✓ Sensor readings muncul
  - ✓ API response 200 OK
- [ ] Refresh aplikasi Vercel, data ESP32 seharusnya muncul

## 🔐 Security Best Practices

- [ ] **Never commit .env.local** (already in .gitignore)
- [ ] **Rotate Supabase keys** setiap 3 bulan
- [ ] **Enable RLS (Row Level Security)** di Supabase untuk tabel
- [ ] **Use service role key hanya di backend** (api/readings.js)
- [ ] **Never expose service key di frontend**
- [ ] **Add HTTPS enforcement** di Vercel settings
- [ ] **Monitor Supabase usage** untuk detect anomalies

---

## 🆘 Troubleshooting

### Aplikasi tidak loading

- [ ] Check Vercel deployment logs
- [ ] Verify environment variables di Vercel dashboard
- [ ] Clear browser cache (Ctrl+Shift+Delete)

### Database connection error

- [ ] Verify Supabase URL dan key di aplikasi
- [ ] Check internet connection
- [ ] Verify tabel `sensor_readings` exist di Supabase
- [ ] Check Supabase firewall/security rules

### ESP32 tidak mengirim data

- [ ] Verify WiFi credentials
- [ ] Check serial monitor untuk error messages
- [ ] Verify Vercel URL adalah benar
- [ ] Test API endpoint manual via Postman:
  ```
  POST https://your-domain/api/readings
  Body: {"device_id":"TPA_01","ch4_ppm":200,"h2s_ppm":10,"temperature":28.5,"humidity":75}
  ```

### Realtime updates tidak working

- [ ] Enable realtime di Supabase dashboard
- [ ] Verify browser supports WebSocket
- [ ] Check browser network tab, look for WebSocket connections
